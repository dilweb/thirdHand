# thirdHand

Telegram-ассистент с памятью, веб-поиском и браузерным агентом.

## Browser Login

Браузерный агент работает в одной Playwright-сессии. Секреты из `.env` не попадают в текст промпта автоматически: модель смотрит снимок страницы и цель; при необходимости спрашивает вас в чате или использует `type_text` с данными из сообщения / контекста.

Что настроить:

- `BROWSER_PROFILE_DIR` — persistent профиль Chromium (cookies остаются между запусками).

Как это работает:

1. Агент открывает сайт в Playwright и читает страницу через `inspect_page`.
2. Поля заполняет через `type_text` из вашего сообщения и контекста; если неясно — спрашивает `ask_user`.
3. При остановке или `ask_user` в Telegram уходит полный текст отчёта отдельными сообщениями и снимок страницы для отладки.

Опционально `BROWSER_SITE_CREDENTIALS_JSON` в `.env` сохранён для совместимости, но агент не вызывает автоподстановку логинов — данные лучше давать в чате.

После смены Dockerfile или compose пересобери контейнеры:

```bash
docker compose down
docker compose up --build
```
